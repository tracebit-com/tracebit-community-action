# AGENTS.md

- Do not edit files under `dist/`. The bundle is generated by `bun run build`.
- `dist/` is in `.gitignore` but tracked in git; use `git add -f dist/` after rebuilding.
- The action has two execution phases: `pre` (dist/pre.js) and `main` (dist/index.js).
  - In sync mode: `pre` issues credentials, writes the AWS profile, and exports env vars. `main` only prints logs.
  - In async mode: `pre` forks a detached child process to issue credentials in the background and writes a temp file path to `_SECURITY_CREDENTIALS_PATH`. `main` reads that file and exports env vars once the background process has
written them.
- `src/run-async.ts` is the child process entry point for async mode.
- Tests are in `src/__tests__/`. Run with `bun run test`.
